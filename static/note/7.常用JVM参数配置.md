# 常用JVM参数配置

> **简介**：本文详细介绍了 JVM 运行时数据区域（程序计数器、虚拟机栈、堆、本地方法栈、方法区）的结构和作用，并提供了两种主流垃圾收集器组合的实战配置方案：**CMS+ParNewGC**和**G1GC**。针对 2 核 4G 容器环境下的 JDK8 应用，给出了完整的参数调优建议、配置示例及日志记录方案，帮助开发者优化内存管理和 GC 性能。


## JVM运行时数据区域详解

1. 程序计数器（Program Counter Register）
  作用：当前线程所执行的字节码的行号指示器  
  特点：线程私有，生命周期与线程相同
  存放数据：
    - 当前线程正在执行的Java方法的字节码指令地址
    - 如果执行的是本地方法，计数器值为空（Undefined）

2. Java虚拟机栈（Java Virtual Machine Stacks）
  作用：存储方法调用的栈帧
  特点：线程私有，可能出现StackOverflowError和OutOfMemoryError
  存放数据：
    - 栈帧（Stack Frame）：每个方法执行时都会创建一个栈帧
    - 局部变量表：存放基本数据类型和对象引用
    - 操作数栈：方法执行时的操作数存储区域
    - 动态链接：指向运行时常量池的方法引用
    - 方法返回地址：方法正常或异常退出的地址

3. 堆（Heap）
  作用：存放对象实例和数组
  特点：线程共享，是垃圾回收的主要区域
  - 存放数据：
    - 所有对象实例：通过new关键字创建的对象
    - 数组对象
    - 字符串常量池（在JDK 7之后移到堆中）  
  - 分区：
    - 新生代（Young Generation）：Eden区、Survivor区（S0、S1）
    - 老年代（Old Generation）
    - 元空间（Metaspace，JDK 8+，属于方法区）或永久代（PermGen，JDK 7及之前）

4. 本地方法栈（Native Method Stack）
  作用：为本地方法（Native Method）服务
  特点：线程私有，与虚拟机栈类似，但服务于本地方法
  存放数据：
  - 本地方法的调用信息
  - 本地方法的局部变量
  - 本地方法的操作数栈

5. 方法区（Method Area）
  作用：存储已被虚拟机加载的类型信息、常量、静态变量等
  实现：JDK 7及之前：永久代（PermGen）占用堆空间，，JDK 8及之后：元空间（Metaspace），使用本地内存
  存放数据：
    - 类型信息：类的完整名称、父类、接口、访问修饰符等
    - 运行时常量池：编译期生成的字面量和符号引用
    - 静态变量：类变量（static变量）
    - 方法信息：方法名称、返回类型、参数、字节码等
    - 即时编译器编译后的代码    


## CMS+ParNewGC（2C4G容器+jdk8）

| 参数                                  | 描述                                                                                          |
|---------------------------------------|-----------------------------------------------------------------------------------------------|
| -Xmn1g                                | 设置年轻代大小为1GB，年轻代包括Eden区和两个Survivor区                                         |
| -Xms3g -Xmx3g                         | 设置初始堆大小为3GB，设置最大堆大小为3GB，设置相同值避免堆大小动态调整的开销                  |
| -XX:+UnlockDiagnosticVMOptions        | 解锁诊断相关的JVM选项，允许使用一些高级的调试和诊断参数                                       |
| -XX:SurvivorRatio=8                   | 设置Eden区与Survivor区的比例，Eden : Survivor = 8 : 1 : 1                                     |
|                                       | 即Eden占年轻代的8/10，每个Survivor占1/10，年轻代1G时：Eden=800MB, Survivor各100MB             |
| -XX:ParallelGCThreads=2               | 设置并行GC线程数为2，用于并行垃圾回收的线程数量                                               |
| -XX:CICompilerCount=2                 | 设置JIT编译器线程数为2，控制即时编译的并行度，在单核或多核环境中都建议至少为2                 |
| -XX:+UseParNewGC                      | 启用ParNew垃圾收集器，用于年轻代的并行垃圾收集，多线程并行回收年轻代对象                      |
| -XX:+UseConcMarkSweepGC               | 启用CMS（Concurrent Mark Sweep）垃圾收集器，用于老年代的并发垃圾收集，尽量减少应用停顿时间    |
| -XX:CMSInitiatingOccupancyFraction=75 | 设置CMS收集器在老年代占用75%时启动，默认约为68%，较高的值减少CMS执行频率，但需要更多剩余空间  |
| -XX:+CMSParallelRemarkEnabled         | 启用CMS重新标记阶段的并行处理，减少重新标记阶段的停顿时间，提高CMS的整体性能                  |
| -XX:+CMSScavengeBeforeRemark          | 在CMS重新标记前先进行一次Minor GC，减少重新标记阶段需要处理的对象数量，从而缩短重新标记的时间 |
| -XX:InitialCodeCacheSize=256m         | 设置CodeCache的初始大小为256MB，CodeCache存储JIT编译后的本地代码                              |
| -XX:ReservedCodeCacheSize=512m        | 设置CodeCache的最大大小为512MB，防止CodeCache无限增长                                         |
| -XX:MinMetaspaceFreeRatio=20          | 设置Metaspace空闲空间最小比例为20%，当空闲空间低于此比例时会触发扩容                          |
| -XX:MaxMetaspaceFreeRatio=80          | 设置Metaspace空闲空间最大比例为80%，当空闲空间超过此比例时会触发收缩                          |
| -XX:MetaspaceSize=256m                | 设置Metaspace的初始大小为256MB，触发首次Metaspace GC的阈值                                    |
| -XX:MaxMetaspaceSize=512m             | 设置Metaspace的最大大小为512MB，防止Metaspace无限增长                                         |
| -XX:MaxDirectMemorySize=512m          | 设置DirectMemory的初始大小为256MB，用于存储NIO缓冲区对象                                      |
| -XX:+UseStringDeduplication           | 启用字符串去重功能（G1GC特有，G1GC中会去除重复的字符串对象）                                  |
| -XX:+PrintGCDetails                   | 打印详细的GC日志信息，包括GC前后各区域的内存使用情况                                          |
| -XX:+PrintGCTimeStamps                | 在GC日志中打印时间戳，显示每次GC发生的相对时间                                                |
| -XX:+PrintGCDateStamps                | 在GC日志中打印日期时间戳，显示每次GC发生的绝对时间                                            |
| -XX:+DisableExplicitGC                | 禁用显式的GC调用，System.gc()调用将被忽略，防止应用程序意外触发Full GC                        |
| -XX:+PrintCommandLineFlags            | 打印JVM启动时的实际参数，包括默认值和用户设置的参数                                           |
| -XX:+HeapDumpOnOutOfMemoryError       | 发生OutOfMemoryError时自动生成堆转储文件，用于分析内存泄漏问题                                |
| -XX:HeapDumpPath=/tmp/dump.hprof      | 指定堆转储文件的保存路径，发生OOM时会在该位置生成dump文件                                     |
| -Xloggc:/tmp/gc.log                   | 指定GC日志文件的路径，所有的GC日志都会写入此文件                                              |
| -XX:+UseGCLogFileRotation             | 启用GC日志文件轮转，防止单个日志文件过大                                                      |
| -XX:NumberOfGCLogFiles=5              | 设置GC日志文件的数量为5个，轮转时最多保留5个日志文件                                          |
| -XX:GCLogFileSize=10M                 | 设置每个GC日志文件的最大大小为10MB，超过此大小时会创建新的日志文件                            |

```bash
java -Dfile.encoding=UTF-8 -jar -Xmn1g -Xms3g -Xmx3g 
-XX:+UnlockDiagnosticVMOptions 
-XX:SurvivorRatio=8 
-XX:ParallelGCThreads=2 
-XX:CICompilerCount=2 
-XX:+UseParNewGC 
-XX:+UseConcMarkSweepGC 
-XX:CMSInitiatingOccupancyFraction=75
-XX:+CMSParallelRemarkEnabled
-XX:+CMSScavengeBeforeRemark
-XX:InitialCodeCacheSize=256m 
-XX:ReservedCodeCacheSize=512m 
-XX:MinMetaspaceFreeRatio=20 
-XX:MaxMetaspaceFreeRatio=80 
-XX:MetaspaceSize=256m 
-XX:MaxMetaspaceSize=512m
-XX:MaxDirectMemorySize=256m 
-XX:+PrintGCDetails 
-XX:+PrintGCTimeStamps 
-XX:+PrintGCDateStamps 
-XX:+DisableExplicitGC
# -XX:+PrintCommandLineFlags
-XX:+HeapDumpOnOutOfMemoryError 
-XX:HeapDumpPath=/tmp/dump.hprof
-Xloggc:/tmp/gc.log
-XX:+UseGCLogFileRotation
-XX:NumberOfGCLogFiles=5
-XX:GCLogFileSize=10M
xxxx.jar > /dev/null &
```


## G1GC（2C4G容器+jdk8）

G1GC典型配置效果：
 - 年轻代回收：并行处理，暂停时间短
 - 老年代回收：增量并发处理，避免长时间STW
 - 内存整理：避免内存碎片问题

| 参数                        | 描述                                                                                                             |
|-----------------------------|------------------------------------------------------------------------------------------------------------------|
| -XX:+UseG1GC                | 启用G1(Garbage First)垃圾收集器，建议堆在4G以上                                                                  |
| -XX:G1HeapRegionSize=8m     | 设置G1垃圾收集器的堆区域大小为8MB                                                                                |
|                             | G1将整个堆划分为多个相同大小的区域(Region)，默认值通常是根据堆大小自动计算的，在3G堆大小下，默认可能是1-32MB范围 |
|                             | Region Size(8MB)：更精细的回收控制，更多的Region数量：3GB/8MB = 384 个Region，更灵活的回收策略                   |
| -XX:MaxGCPauseMillis=200    | 设置G1垃圾收集器的最大暂停时间，单位为毫秒，默认值为200毫秒                                                      |
| -XX:G1NewSizePercent=20     | 设置年轻代大小占比，默认值为25                                                                                   |
| -XX:G1MaxNewSizePercent=40  | 设置年轻代最大占比，默认值为40                                                                                   |
| -XX:ParallelGCThreads=2     | 设置并行GC线程数为2 影响：并行标记阶段的线程数、并行清理阶段的线程数、Eden区分配的线程数                         |
| -XX:+UseStringDeduplication | 启用字符串去重功能（G1GC特有），在并发标记阶段识别重复的字符串并合并，只对相同内容的字符串生效                   |
|                             | 效果：减少堆内存使用、特别适合处理大量文本数据的应用                                                             |


```bash
java -Dfile.encoding=UTF-8 -jar -Xms3g -Xmx3g 
-XX:+UseG1GC 
-XX:MaxGCPauseMillis=200    
-XX:G1HeapRegionSize=8m     
-XX:G1NewSizePercent=20     
-XX:G1MaxNewSizePercent=40  
-XX:ParallelGCThreads=2 
-XX:CICompilerCount=2 
-XX:InitialCodeCacheSize=256m 
-XX:ReservedCodeCacheSize=512m 
-XX:MetaspaceSize=256m 
-XX:MaxMetaspaceSize=512m 
-XX:MaxDirectMemorySize=256m 
-XX:+UseStringDeduplication 
-XX:+PrintGCDetails 
-XX:+PrintGCTimeStamps 
-XX:+PrintGCDateStamps 
-XX:+DisableExplicitGC 
-XX:+HeapDumpOnOutOfMemoryError 
-XX:HeapDumpPath=/tmp/dump.hprof
-Xloggc:/tmp/gc.log
-XX:+UseGCLogFileRotation
-XX:NumberOfGCLogFiles=5
-XX:GCLogFileSize=10M
xxxx.jar > /dev/null &
```
